设计一个方便 c 解析的，db 语言：

- `put path value` 创建或修改 record 数据。
- `add path value` 给 list 增加数据。

先不设计删除数据的功能，如果需要可以考虑：

- `del path` 删除 record 数据。

```linn
put name "Alice"
put user/age 30
add tags "a"
add tags "b"
put matrix/0/0 1
put matrix/0/1 2
```

```json
{
  "name": "Alice",
  "user": {"age": 30},
  "tags": ["a", "b"],
  "matrix": [[1,2]]
}
```

需要空 record 和 list 时，可以用 `{}` 和 `[]`：

```linn
put name "example"
put stats/count 42
put stats/empty_obj {}
put logs []
put matrix/0/0 1
put matrix/0/1 2
put matrix/1 []
```

```json
{
  "name": "example",
  "stats": {"count": 42, "empty_obj": {}},
  "logs": [],
  "matrix": [[1,2], []]
}
```

# stack vm

通过扩展出来一个 `ins` 操作，可以取代传统汇编语言。
每次定义一个 instruction 都要带上所属于的函数。

以某个 stack vm 为例：

```scheme
(define (factorial n)
  (if (int-less-or-equal? n 1)
    1
    (imul (factorial (isub n 1)) n)))
```

```linn
ins factorial   local-store n
ins factorial label body
ins factorial   local-load n
ins factorial   literal 1
ins factorial   call int-less-or-equal?
ins factorial   jump-if-not recur-case
ins factorial   jump base-case
ins factorial label base-case
ins factorial   literal 1
ins factorial   return
ins factorial label recur-case
ins factorial   local-load n
ins factorial   literal 1
ins factorial   call isub
ins factorial   local-store n1
ins factorial   local-load n1
ins factorial   call factorial
ins factorial   local-store f1
ins factorial   local-load n
ins factorial   local-load f1
ins factorial   tail-call imul
```

可以利用数据库机制来增加 metadata：

```linn
put factorial/arity 1
```
